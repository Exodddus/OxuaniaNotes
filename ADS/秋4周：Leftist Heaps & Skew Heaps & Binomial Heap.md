# Leftist Heaps

定义**null path length** ：Npl(X)是节点X到没有孩子的节点的最短路径长度，定义Npl(NULL)=-1
Npl(X) = min{ Npl(C) + 1 for all C as children of X }

leftist heap property: 对于堆中的每个节点，其左孩子的空径长不小于右孩子的空径长。

定理：对于一个左偏堆，如果它右侧的路径上有 $r$ 个节点（即根节点的空指针路径长度为 $r−1$），那么它至少有 $2^r-1$ 个节点。
可以使用归纳法给出证明

# Skew Heaps
和左偏堆的关系类似AVL树和SplayTree的关系，去除了`Npl`字段，但是M次连续操作消耗至多$O(MlogN)$的时间。
选择拥有更小的根节点的树作为被附加树，拆开它的右子树，然后把它的左子树换到右子树的位置上，再对它的（原）右子树和附加树进行递归比较。也就是说每一次的附加，都需要一次交换操作。

# 二项堆

**二项堆**(binomial heap) 不是单个的满足堆序（即根节点的值是整棵树的最值）的树，而是一个满足堆序的树的集合，即**森林**；而其中每棵满足堆序的树被称为**二项树**(binomial tree)，它有如下递归定义：

- 高度为 0 的二项树是一棵单节点的树
- 高度为 $k$ 的二项树 $B_k$​ 由两棵高度为 $k−1$ 的二项树 $B_{k−1}$​ 构成，且其中一棵树附着在另一棵树的根节点上![[BinomialHeap示意图.png]]
因此，可以用二项堆唯一表示任意大小的堆。
